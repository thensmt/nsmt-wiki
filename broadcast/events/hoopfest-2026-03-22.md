# Hoopfest — 2026-03-22

**Venue:** TBD (first NSMT broadcast — venue not recorded in source data)
**Crew:** Two-person setup: one on iPad (primary controller), one on MacBook Pro producer (backup)
**Stream:** First live NSMT broadcast via [Castr](../../systems/castr.md)

## Crew Roles

| Position | Device | Responsibilities |
|----------|--------|-----------------|
| Primary Controller | iPad | Scores, clock, timeouts, fouls, player stats, spotlight |
| Backup / Producer | MacBook Pro | Backup control, graphic overlays, ticker |

## What Went Well
- Successfully completed the first-ever NSMT live broadcast
- Proved the core production workflow: [cameras](../equipment/cameras.md), [Teradek wireless](../equipment/video-transport.md), [YoloBox switching](../equipment/video-transport.md), [Elgato capture](../equipment/video-transport.md), [OBS compositing](../../systems/obs-configuration.md), RTMP streaming
- Scorebug overlay system worked in a live environment
- **Player spotlight card** worked — photo + stats popup displayed on overlay during broadcast
- **iPad as primary game controller** proved viable for live game operation

## What Went Wrong
- [Hollyland Solidcom SE](../equipment/comms.md) intercom batteries died during the broadcast
  - Production crew lost communication with camera operators mid-event
  - No spare batteries were available on-site

## Lessons Learned
- **Spare batteries are mandatory.** The Hollyland headset battery failure was the biggest equipment issue. Action: source spare/replacement batteries for all future events. Updated in [Comms](../equipment/comms.md) and [Power](../equipment/power.md).
- **Build a complete battery checklist** for pre-game setup that includes checking charge level on every battery-powered device. Updated in [Pre-Game Runbook](../runbooks/pre-game.md).
- **Debrief after every event** to capture lessons while they're fresh. This report is the template for future events.
